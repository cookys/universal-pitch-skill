## 1. Michael Seibel's Seven Questions (YC Seed Pitch)

Answer these seven in order. If the answers are clear, the pitch writes itself.

| # | Question | What Good Looks Like | Common Mistake |
|---|----------|---------------------|----------------|
| 1 | What do you do? | Company name + one sentence + one example | Abstract jargon ("We leverage AI to...") |
| 2 | How big is the market? | Transparent math (units × price) | Quoting Gartner without logic |
| 3 | What's your traction? | Numbers WITH time frame | "We have early customers" (no context) |
| 4 | What's your unique insight? | What you know that others don't | "We use AI" (not unique) |
| 5 | What's your business model? | How you charge, validated or assumed | Avoiding the money question |
| 6 | Who's your team? | Specific achievements | Job titles without substance |
| 7 | What do you want? | Exact amount + 18-month milestones | Vague "we're raising a round" |

**Email Test**: Write your one-liner. Send it to a smart friend. Ask them to explain it
back. If they ask any clarifying question, rewrite until they don't.

**"Make it sound dumber than you think it should."** — Michael Seibel

## 2. JTBD — Jobs to Be Done Template

### Format

```
When I [situation/trigger],
I want to [motivation/action],
So I can [expected outcome].
```

### Three Layers

| Layer | Question | Example |
|-------|----------|---------|
| Functional | What task are they doing? | "Review my team's code contributions weekly" |
| Social | How do they want to be seen? | "Be the manager who uses data, not gut feeling" |
| Emotional | How do they want to feel? | "Confident my performance reviews are fair" |

### Discovery Questions (from Mom Test)

Don't ask "would you use this?" — ask about their current reality:

1. "Tell me about the last time you had to [do the job]. What happened?"
2. "How are you solving this today? Walk me through it step by step."
3. "What's the most annoying part of that process?"
4. "How much time/money does this cost you per week/month?"
5. "Have you tried to fix this before? What happened?"
6. "If you could wave a magic wand, what would change?"

**Gold**: When the same pain language appears in 3+ interviews, use their exact words in
the pitch. "My customers told me they spend 5 hours a week on this" beats any market
analysis.

## 3. Value Proposition Canvas

```
┌─────────────────────────────────────────────────┐
│                  YOUR PRODUCT                    │
│                                                  │
│  Products & Services    Pain Relievers           │
│  ┌──────────────┐      ┌───────────────┐        │
│  │ What you     │      │ How you       │        │
│  │ offer        │      │ remove pains  │        │
│  └──────────────┘      └───────────────┘        │
│                                                  │
│  Gain Creators                                   │
│  ┌───────────────┐                               │
│  │ How you       │                               │
│  │ create gains  │                               │
│  └───────────────┘                               │
└─────────────────────────────────────────────────┘
        ↕ FIT = alignment between left and right
┌─────────────────────────────────────────────────┐
│                  YOUR CUSTOMER                   │
│                                                  │
│  Customer Jobs          Pains                    │
│  ┌──────────────┐      ┌───────────────┐        │
│  │ What they    │      │ What blocks   │        │
│  │ try to do    │      │ or frustrates │        │
│  └──────────────┘      └───────────────┘        │
│                                                  │
│  Gains                                           │
│  ┌───────────────┐                               │
│  │ What success  │                               │
│  │ looks like    │                               │
│  └───────────────┘                               │
└─────────────────────────────────────────────────┘
```

**Pitch translation**: Problem slide = right side (Customer). Solution slide = left side
(Product). Product-Market Fit = tight alignment between them.

Every Pain Reliever must map to a specific Pain. Features without corresponding pains
don't belong in the pitch.

## 4. Mom Test — Key Rules

From Rob Fitzpatrick's "The Mom Test":

**Rule 1**: Talk about their life, not your idea.
- Bad: "Would you use a tool that...?"
- Good: "How do you handle X today?"

**Rule 2**: Ask about specifics in the past, not generics about the future.
- Bad: "Would you pay for this?"
- Good: "How much did you spend last time you tried to fix this?"

**Rule 3**: Talk less, listen more.
- Bad: Pitching for 10 minutes, then asking "thoughts?"
- Good: "Tell me more about that. What happened next?"

**Converting interviews to pitch language**:
- Count how many people mention the same pain → this becomes your Problem slide
- Use their exact words → more authentic than marketing speak
- Note the workarounds they've built → these become your "Before" state

## 5. Selling Point Distillation Examples

### Example A: Code Scanning Feature

```
Layer 1 (Feature → Capability):
  "Daily GitLab commit scanning with 26-language AST analysis"
  → "Automated code contribution tracking across any language"

Layer 2 (Capability → Emotional Outcome):
  → "Never worry about missing someone's contributions"

Layer 3 (Emotional → Narrative Hook):
  Contrast: "Before: 3 hours reading commits. After: 5-minute dashboard scan."
  Quantify: "Reviews 100,000+ lines/day. A human would need 2 weeks."
  Social:   "Every engineer's work is visible to the whole team."
```

### Example B: Gamification System

```
Layer 1: "EXP system with 6-dimension skill tree and seasonal leaderboard"
  → "Game mechanics that make coding practice addictive"

Layer 2: → "Feel like you're leveling up, not just clocking hours"

Layer 3:
  Contrast: "Before: coding feels invisible. After: every commit earns EXP."
  Quantify: "New hires reached senior-level EXP in 8 weeks of high output."
  Social:   "Leaderboard makes friendly competition natural."
```

### Example C: AI Personality System

```
Layer 1: "9 AI personalities for code review feedback"
  → "Customizable AI mentor that reviews code in your style"

Layer 2: → "Code review stops being scary and starts being fun"

Layer 3:
  Contrast: "Before: cold automated feedback. After: your AI cat roasts your code."
  Quantify: "Code review engagement up 340% after enabling AI personalities."
  Social:   "Your AI pet battles colleagues' AI pets. The whole office watches."
```

## 6. Competition Positioning (2x2 Matrix)

Never say "we have no competitors." Investors hear: "we haven't done our homework."

### How to Build a 2x2

1. Choose TWO axes where you're strong and alternatives are weak
2. Place yourself in the top-right quadrant
3. Name the categories, not specific competitors (unless well-known)

```
Example axes for a dev tools pitch:

                    AI-Native Analysis
                         ▲
                         │
            ┌────────────┼────────────┐
            │            │     ★ You  │
            │ "Generic   │            │
            │  APM tools"│            │
    Manual ─┼────────────┼────────────┼─ Gamified
            │            │            │
            │ "Spreadsh- │  "Basic    │
            │  eet track-│   dashb-   │
            │  ing"      │   oards"   │
            └────────────┼────────────┘
                         │
                    Manual Review
```

### What Goes in Each Quadrant

- **Top-right (you)**: Your product name
- **Bottom-left**: "The old way" (manual, spreadsheets)
- **Other quadrants**: Categories of alternatives (not specific company names unless they're household names)
